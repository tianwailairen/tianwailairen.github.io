Flex布局是什么？

Flex 意为“弹性布局”,用来为盒状模型提供最大的灵活性。任何一个容器都可以指定Flex布局，块元素可以，行内元素也可以。
Webkit内核的浏览器，必须加上-webkit 前缀。
例如：
.box{
display:-webkit-flex; /Safari/
display:flex;
}

基本概念

1 flex 容器（flex container）：采用Flex布局的元素。
2 flex 项目： 容器的所有子元素自动成为容器成员，简称为项目
3 水平主轴（main axis）:主轴开始的位置叫main start,结束的位置叫main end
4 交叉轴（cross start ）:交叉轴的开始位置叫做 cross start,结束的位置叫cross end

容器的属性

  • flex-direction 用处：决定主轴的方向。也就是项目的排列方向。
    值： 

    row（默认值）：主轴为水平方向，起点在左端。
row-reverse：主轴为水平方向，起点在右端。
column：主轴为垂直方向，起点在上沿。
column-reverse：主轴为垂直方向，起点在下沿。

  • flex-wrap用处： 如果一条轴线排不下，如何换行
    值：

    nowrap(默认)：不换行
wrap: 换行，第一行在上方
wrap-reverse: 换行，第一行在下方

  • flex-flow用处：定义了项目在主轴上的对齐方式。
    值：

    flex-start(默认值):左对齐
flex-end:右对齐
center:居中
space-between:两端对齐，项目之间的间隔相等
space-around:每个项目两侧的间隔相等，所以项目之间的间隔比项目与边框的间隔大一倍

  • align-items 用处定义项目在交叉轴上如何对齐
    值：

    stretch(默认值): 如果项目未设置高度或设为auto,将占满整个容器
flex-start: 交叉轴的起点对齐
flex-end: 交叉轴的终点对齐
center:交叉轴的中点对齐
baseline:项目的第一行文字的基线对齐

  • align-content 用处定义了多根轴线的对齐方式。如果项目中只有一根轴线，该属性不起作用。
    值：

    flex-start：与交叉轴的起点对齐。
flex-end：与交叉轴的终点对齐。
center：与交叉轴的中点对齐。
space-between：与交叉轴两端对齐，轴线之间的间隔平均分布。
space-around：每根轴线两侧的间隔都相等。所以，轴线之间的间隔比轴线与边框的间隔大一倍。
stretch（默认值）：轴线占满整个交叉轴。

项目的属性

  • order用处定义项目的排列顺序。数值越小，排列越靠前，默认为0；
    值：

  • flex-grow用处定义项目的放大的比例，默认为0，即如果存在剩余空间，也不放大。
    （如果所有项目的flex-grow属性都为1，则它们将等分剩余空间（如果有的话）。如果一个项目的flex-grow属性为2，其他项目都为1，则前者占据的剩余空间将比其他项多一倍。）

  • flex-shrink 用处定义了项目的缩小比例，默认为1，即如果空间不足，该项目将缩小。
    （如果所有项目的flex-shrink属性都为1，当空间不足时，都将等比例缩小。如果一个项目的flex-shrink属性为0，其他项目都为1，则空间不足时，前者不缩小。）

  • flex-basis定义了在分配多余空间之前，项目占据的主轴空间。浏览器根据这个属性，计算主轴是否有多余空间。它的默认值为auto,即项目的本来大小。
    （它可以设为跟width或height属性一样的值（比如350px），则项目将占据固定空间。）

  • flex 属性是 felx-grow, flex-shrink, flex-basis的简写，默认值为0 1 auto。 后两个属性可选。
    （该属性有两个快捷值：auto (1 1 auto) 和 none (0 0 auto)。
    建议优先使用这个属性，而不是单独写三个分离的属性，因为浏览器会推算相关值。）

  • align-self用处：允许单个项目有与其他项目不一样的对齐方式，可覆盖align-items属性。默认值为auto,表示继承父元素的align-items属性，如果没有父元素，则等同于stretch。

gulp 介绍

gulp是什么？

gulp是一个基于流的构建系统，可以自动执行指定的任务
可以自动完成javascript/coffee/sass/less等文件的测试，检查，合并，压缩，格式化，部署文件生产等，同时可以监听文件改动后重复指定的这些步骤

gulp原理 & 流程

gulp能做什么

  1. 合并css/javascript等
  2. 压缩html/css/javascript等
  3. 合并精灵图/iconfont等
  4. 预编译less,sass,coffeescript，es6等
  5. 使用postcss等工具，比如uncss/autoprefixer
  6. 压缩jpg/png图片
  7. 使用条件编译输出不同的结果，比如app页面和mobile页面
  8. 实时重载/css注入，提高开发速度
  9. 打包生成线上版本代码
  10. 等等…

安装gulp

配置文件 gulpfile.js

5个方法

  1. task
  2. run
  3. watch
  4. src
  5. dest
